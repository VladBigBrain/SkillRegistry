# User Story (US)

## Цель

Зафиксировать **ценность** и **ожидаемый результат** с точки зрения пользователя, а не техническую реализацию.

## Вход (если не хватает — задай 3–7 коротких вопросов)

- Роль: кто пользователь/персона?
- Действие: что хочет сделать?
- Выгода/результат: зачем, какая ценность?
- Контекст/сценарий: где и при каких условиях?
- Границы: что **входит** и что **не входит** в US?
- Исключения/риски: частые ошибки, ограничения, крайние случаи?
- Метрики успеха: как понять, что «сработало»?

Если пользователь не знает ответы — явно зафиксируй **допущения** и вынеси **открытые вопросы** в конец.

## Выход (строго в этом порядке)

### 1) Заголовок

Коротко: действие + объект + контекст (если нужен).

### 2) User Story (1 предложение)

**Как [роль], я хочу [действие], чтобы [выгода].**

### 3) Описание (2–6 буллетов)

Опиши наблюдаемое поведение/результат: что пользователь видит/получает, без реализации.

### 4) Критерии приёмки (Acceptance Criteria) — Given/When/Then

- **AC1:** Given … When … Then …
- **AC2:** Given … When … Then …

Правила:
- Пиши проверяемо (однозначный результат).
- Покрой «happy path» + 1–3 важных edge cases.
- Не включай технические решения (БД, очереди, фреймворки), только поведение.

### 5) Тестовые сценарии (Test Scenarios)

Для каждого сценария укажи:
- **Предусловия**
- **Шаги**
- **Ожидаемый результат**

Минимум: 1 позитивный + 1 негативный/пограничный.

### 6) Допущения и открытые вопросы (если есть)

Список, который надо уточнить у стейкхолдеров.

## Мини-пример (кинотеатр)

### Заголовок
Выбор места на сеансе

### User Story
Как зарегистрированный пользователь, я хочу выбрать свободное место на выбранный сеанс, чтобы купить билет на удобное место.

### Критерии приёмки (Given/When/Then)
- AC1: Given пользователь авторизован и выбрал фильм/дату/сеанс When открывает схему зала Then система показывает только доступные места и позволяет выбрать одно место.
- AC2: Given место стало недоступным (кто-то купил) When пользователь пытается подтвердить выбор Then система сообщает об ошибке и предлагает выбрать другое место.

### Тестовые сценарии
### TS1: Успешный выбор
- Предусловия: пользователь авторизован; выбран сеанс; есть доступные места.
- Шаги: открыть схему зала → выбрать место → подтвердить.
- Ожидаемый результат: переход к оплате/оформлению с выбранным местом.

### TS2: Место занято в момент подтверждения
- Предусловия: пользователь авторизован; выбран сеанс; выбранное место становится занятым.
- Шаги: выбрать место → подтвердить.
- Ожидаемый результат: показ сообщения; место снимается; доступен выбор другого.
